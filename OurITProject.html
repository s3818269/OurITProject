  • Scope and Limits

Our desirable goal are the livestreams videos reach to full HD resolution. Live chat that streamers can interact to the viewers and connection is stable during the stream. Our website is also expected to have bandwidth for 100,000 viewers access at the same period of time. Due to limited time and resources, we can not achieve the goal that we expected. After next 2 month, we predict that our video quality can only reach 480p at max. Moreover, it is not guaranteed that the connection will not be interrupted during the whole livestream. We still do not create the live chat function. For the bandwidth issues, for now we can only afford for 30,000 to 40,000 people to join the stream. 

  1. Tools and Technologies

There are countless tools to create a livestream platform like Twitch, NimoTv, NoNo Live. Each tool has a different programming method and different algorithms that are suitable for many types of website. For our project, we decided to use Wowza Streaming Engines tools to build up the livestream app on a web platform. Wowza Streaming Engine is also known as Wowza Media Server prior to version 4. Developed and created by Wowza Media Systems.

Wowza is a unified live media server software. Use the server to stream videos and play back videos, audio on demand via IP network to transmit to computer devices such as PCs, laptops. In addition, the server is developed based on JavaScript programming. This software is asking for licenses, besides that, there will be a 30-day free trial for developers. However, this free trial version can only provide the basic functionality to create a livestream platform. But to be able to develop more deeply and diversified, this application requires developers to purchase for the licenses and get the latest Wowza version. Wowza Streaming Engine 4.8.5 is the latest authorized version for developers. In order to run and maintain the website stably, we need hardware devices able to work at the system server. We choose a workstation-oriented computer configuration which has a CPU and VGA that is strong enough to be able to work multi task functions, such as encode videos, and host a server. In order to host a server and record livestream, we use Wowza Streaming Cloud version 1.3. Wowza player 1.1.21 build 2 is an excellent choice for the developers to upload the HD quality videos and deliver them to the users. 

  1. Testing

There are two steps to test our project, which are code testing and functionality testing. First of all, our team is using a code checking tool called SONAR, this tool will compare our code with the standard code then check if there are any coding violations. After that, upload all the code through GitHub in order to receive feedback from peers developers in order to optimize the code. Secondly, we test the function of our project by broadcasting a testing livestream and see the outcome. Furthermore, to test if our program can run on different kinds of devices we create the early access version of our project and ask the streamers for testing the new platform. We plan to ask approximately 100 streamers from many regions around the world to test our product and give some feedback to us. In exchange, we will pay a certain amount of money for them. Our expected outcome is the videos have HD quality or higher and connection not being interrupted.

  1. Risks

There are several risks that our team might encounter during the project. Firstly, our programming, coding skills just reach the amatuer level at the moment. That might lead to some difficulties while we code and fìnd the bugs, as well as find the solution to debug. One thing that we are concerned about is that all the computer components running in the workstation cannot operate the entire system smoothly because the program requires a more powerful configuration that our computers cannot respond to even though it is running at full capacity and leads to CPU, VGA, RAM throttling. The other risk that may happen is the financial problem. During the project, the price of hardware components that were used to build the workstation can increase, as well as the streamers who we ask to test the products may want more money than we expect. This can make our fund not enough to support our project in the long term.

  1. Skills and Jobs

As a project manager, 4 positions that we need to recruit in order to develop this project are Information Architect, Web Developer, User Interface (UI) Designer and Finance Manager. Firstly for the Web Developer position, this job is in charge of coding, designing and layout a website based on the group manager’s demand. The web developer is also required to have experience in programming languages such as JavaScript, C++, HTML, database systems. Secondly, the UI Designer is responsible for creating the concept of the user experience, ensuring all interactions are intuitive and easy for customers to use, as well as improve the appearance of the software. In addition, having a skill on various visual design programs and some soft skills such as problem solving skills and teamwork is the must. Thirdly, for the Finance Manager, this job is in charge of planning the budget and providing the financial advice to the team leader. The requirements for this position are negotiation skills, good communication skills, keen eye for detail. Lastly, the Information Architect’s missions are to plan and design the information structure for websites and web applications. This job is required to have some experience in web production softwares such as Visio, Dreamweaver, Photoshop.
